\chapter{CONCLUSION \& FUTURE WORK}
\label{ch:conclusion}

Cryptographic implementations form the practical boundary between formal security guarantees and deployed systems. A protocol may be mathematically sound, but its real-world security depends on whether the implementation correctly realizes the intended computation, whether security-critical cryptographic code can be identified, and whether testing methods can expose implementation-level errors before deployment. The works presented here advance these goals by developing systematic evaluation, automated checking, and binary-level testing techniques for cryptographic software.

The study of cryptographic function identification provides a comprehensive evaluation of existing approaches for detecting cryptographic algorithms in binaries. We conduct reproduction and replication studies, organize the design space of existing tools, and develop a unified benchmarking framework that supports more consistent comparison across approaches. The evaluation covers modern cryptographic algorithms, microbenchmarks, libraries, and large applications, while also accounting for practical variations introduced by compilers, optimization levels, obfuscation strategies, operating systems, and algorithm versions. This work clarifies the current state of cryptographic function identification and provides a benchmark foundation for evaluating future detection techniques in a more systematic and reproducible way.

SNARKProbe advances automated analysis for zkSNARK implementations by targeting cryptographic logic errors that are difficult to detect through ordinary testing. Instead of only checking whether a program crashes or whether a proof can be generated, SNARKProbe checks whether the implementation is consistent with the intended proof statement. It combines constraint checking, fuzzing-based testing, value monitoring, and error localization to inspect both proof programs and zkSNARK libraries. By helping identify inconsistencies across the constraint system, witness generation, and verification logic, SNARKProbe reduces the manual effort required to analyze complex zkSNARK implementations and provides a practical framework for improving confidence in zero-knowledge proof software.

The grey-box differential testing framework extends implementation analysis to zero-knowledge proof binaries. This work addresses the deployed setting where source-level assumptions may be insufficient and where the executable itself must be tested. The framework uses the expected cryptographic statement as the oracle, identifies security-relevant execution behavior, and localizes inconsistencies across the major components of the proof workflow. By combining structured input handling, cryptographic-input-aware coverage analysis, control-dependency-aware taint tracking, and differential checking, the framework moves binary testing beyond general crash detection and toward detecting logic-level errors in zero-knowledge proof applications.

As cryptographic systems continue to support increasingly complex applications, their security must be evaluated not only through protocol design but also through implementation behavior. The contributions presented here show that practical cryptographic security benefits from analysis methods that are systematic, automated, and applicable to real software artifacts. By improving how cryptographic code is evaluated in binaries, how zero-knowledge proof implementations are checked, and how deployed proof applications are tested, this work helps close the gap between formal cryptographic guarantees and the software systems that rely on them.
